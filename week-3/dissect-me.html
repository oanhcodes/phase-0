Introduction to Version Control, Git, and GitHub

December 6, 2015

What is version control? Version control is system that keeps track of your file(s) and all the changes you’ve made to them over time. This means that you are able to return to a previous version of your work at any time. Mess something up? Well, fear not! With version control, you can just jump back to an earlier version and start working again. Cool right?

Instead of keeping track of all the changes you make individually, version control takes all the changes you’ve made and and bundles it into a something called a commit. A commit is a similar to save point or snapshot. It is a place you can always return to in your work.

Version control is a great way to keep track of individual work flow, but where it really shines is in a collaborative setting. By using version control, it is possible for multiple people to make edits and changes to a file at the same time without overwriting each other’s work. This is possible because version control merges two or more commits together into a new document.

Git is a version control system you can download that is popular with developers. It is accessed from your computer’s terminal so you will need to be familiar with how to use the command line. The command line is a way to navigate and browse your computer using text-only commands that is more efficient that manually clicking and searching for files. Before using git, it’s a good idea to go over basic command line commands.

To save and keep track of your changes using git, you use basic commands like add and commit. One of the great features of git is that you can create and work in “branches”. The changes you make and test in branches won’t effect the “trunk”, the master version or your work, until you’re ready to merge all your changes.

It is important to note that git lives in locally in your computer and that is is different from GitHub, which is a remote repository. A remote repository is a place where your directories and files can live on the web. If you want to share your files and changes with other people, you will need to push it to a GitHub or a similar remote repository. By sharing to a remote repository like GitHub, programmers can share their code and start collaborating.

So let’s wrap this up. 1. Version control is a way to keep track of your files and the changes made to it. 2. Git is a popular version control software that lives in your computer locally. You can use it to keep track of changes by making commands in your computer’s terminal. Git allows you to work in branches so you can test your code before merging it into your master code. 3. Github is a remote repository where you can push and pull code. It is a place where people can share their code and collaborate. Happy coding! -Linda
